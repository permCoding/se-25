# labrab-03 + ladrab-04  

## Классы в JS  

> классы, наследование, инкапсуляция, полиморфизм  

### Задание  

Разработать **систему управления библиотекой книг**.  

**Описание задания**:  

- реализовать консольное Node.js-приложение с классами для моделирования процессов в библиотеке  
- данные хранить в csv или json (на ваш выбор)  

Основные классы (минимально необходимый набор):  

- Book (с приватными полями id, title, author, isbn),  
- User (с полями Фамилия, Имя, Номер_карточки и с методами getBook и returnBook),  
- Library (управляет массивом книг, массивом пользователей и производимыми операциями)  

Продумайте как лучше разбить на таблицы, хранить и обрабатывать данные.  
Следует учесть, что для одного ISBN может быть несколько экземпляров книги.  
Когда пользователь получает книгу, количество экземпляров уменьшается.  

```txt
у всех одинаковых книг ISBN одинаковый  
International Standard Book Number -  это международный стандартный книжный номер
```

Предполагаемые методы работы с книгами:  

- addBook (title, author, isbn, amount),  
- delBook,  
- findBook (в том числе сколько осталось экземпляров),  
- issueBook (выдать книгу пользователю),  
- getBooks (вернуть список книг - название, автор, ISBN, количество экземпляров общее и сколько на выдаче)  

Для работы с пользователями аналогично примерно так - добавить/удалить пользователя, вернуть список пользователей, книги у пользователя, ...  

По необходимости можете добавлять свои методы или классы.  

Для выполнения задания требуется применять:  

- геттеры/сеттеры,  
- статические методы (например, для валидации ISBN),
- наследование (например, класс Admin от User с дополнительными правами),  
- приватные поля (#title, #books) для инкапсуляции,  
- конструкторы с параметрами и валидацией,  
- обработка ошибок через try-catch в методах классов.  

**Тестировать** работоспособность через `консоль` с передачей параметров в строке и получением через `process.argv` (для интерактивного меню: добавить книгу, выдать, найти).  

В дальнейшем можно будет перенести классы:  

> перенести в web-app  
> данные хранить в SQL  

Небольшой пример структуры кода:  

```js
class Book {
    #title;
    constructor(title, author, isbn) {
        this.#title = title;
        // ...
    }
    getTitle() { return this.#title; }
}

class Library {
    #books = [];
    addBook(book) { /* ... */ }
}
```

Пример запуска в консоли:  
`node library.js add "Война и мир" Толстой 12345 2`  

> добавили в библиотеку `две` книги `Толстого` "Война и мир" с ISBN `12345`  
> при запуске программы читаются файлы с данными, исполняется задание, результат выводится в консоль, изменения сохраняютсмя в файлах
