### Примитивные типы данных  

- хранят только одно значение  
- неизменяемы  
- передаются `по значению`  
- сравниваются непосредственно  

1. **number** — целые числа и числа с плавающей точкой  
2. **string** — строки (текст)  
3. **boolean** — логические значения (`true` или `false`)  
4. **undefined** — означает, что значение не присвоено  
5. **null** — означает «ничего» (задаётся разработчиком)  
6. **symbol** — уникальный идентификатор  
7. **bigint** — для очень больших целых чисел  

### Составные типы данных  

- коллекции данных  
- сложные структуры  
- передаются `по ссылке`  
- НЕ сравниваются непосредственно  

**Object** — базовый составной тип, от него происходят остальные:  

- **Array** (массив)  
- **Function** (функция)  
- **Date** (дата)  
- **RegExp** (регулярное выражение)  
- **Map**, **Set**, **WeakMap**, **WeakSet** (коллекции)  

---  

```js
null == undefined;  // true  (нестрогое сравнение, равны)
null === undefined; // false (строгое сравнение, разные типы)  
```

> undefined: "Значение еще не установлено"  
> null: "Ссылка указывает в никуда" (преднамеренно так установлено)  

---  

```js
Number(undefined); // NaN (Не число)
Number(null);      // 0
```

```javascript
Number(undefined); // NaN (Не число)
Number(null);      // 0
```

**`undefined`** используется для проверки, существует ли свойство или было ли присвоено значение  
**`null`** используется для "обнуления" переменной или явного указания, что объекта нет (ПОКА нет, но потом планируется туда записать новый объект)
