## Простое серверное приложение

Серверное приложение запускается на стороне сервера и ждёт от клиента запрос. Приложение крутится в бесконечном цикле - ждёт запроса, после получения обрабатывает его и посылает соответсвующий ответ клиенту в виде **htlm-страницы**, **json-файла** и т.п.  

Программный интерфейс приложения - совокупность правил (видов запросов), выполненных по стандарту (REST API, GraphQL, RPC), которые это приложение поддерживает.  

Например в REST API:

- GET-запрос - позволяет получить что-то от сервера
- POST-запрос - позволяет передать что-то на сервер

Запрос выглядит просто как строка - `http-адрес`.  
В строке адреса можно передать дополнительные параметры, например:  

```js
http://localhost:3000/getByNumber?number=23  
http://localhost:3000/sortBy?direct=desc  
http://localhost:3000/sortBy?field=id&direct=asc  
```

Эндпойнт (endpoint) - это (адрес+метод HTTP) по которому к веб-сервису (приложению) можно обратиться для выполнения определенных действий (POST, PUT) или получения данных (GET).
К серверному приложению может обратиться клиент: браузер, мобильное приложение, другой сервер.  

Характеристики эндпойнта:  

- URL — адрес (например, `/api/users`)  
- HTTP метод — действие (GET, POST, PUT, DELETE)  
- параметры — данные для обработки (в URL, заголовках или теле запроса)  
- ответ — что возвращает сервер (JSON, HTML, файл)  

```js
GET  /products
POST /products
```

### Создаём справочник  

- Пользователь посылает запрос через адресную строку браузера.  
- Приложение выводит на экран html-страницу.  
- Данные хранятся в json-файле.  
- Будем делать только GET-запросы (json-файл пользователь не будет менять).

В Node.js `__dirname` - это специальная глобальная переменная, которая содержит **абсолютный путь** к директории, в которой находится текущий исполняемый файл. Для статических файлов и других ресурсов приложения удобнее использовать именно `path.join(__dirname, ...)`. Использование `path.join(__dirname, 'public', 'favicon.ico')` имеет несколько важных преимуществ по сравнению с относительным путем `"./public/favicon.ico"`:

1. **Кросс-платформенность**: метод `path.join()` обрабатывает различия в разделителях путей между операционными системами (например, `/` для Unix-подобных систем и `\` для Windows), что делает код переносимым.  

2. **Надежность**: если запускать код из другой директории (например, из терминала, находясь в другом каталоге), то относительный путь может не найти файл.

3. **Абсолютный путь**: Использование `__dirname` помогает гарантировать, что даже при смене названия рабочего каталога не возникнет ошибки при обращении к файлам.
