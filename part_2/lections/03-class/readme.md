---  

ООП делится на два вида:  

- классно-ориентированное  
  `Class-based` - C++, C#, Java, Python, PHP, Swift  
- прототипно-ориентированное  
  `Prototype-based` - JavaScript, Lua  

В классно-ориентированном (обобщённо):  

- класс является абстрактным шаблоном  
- используя который создаются объекты (инстанцируются, экземпляр класса)  
- классы могут находиться в отношениях `родитель-потомок`  
- статичная иерархия классов  
- наследуются свойства, методы  
- обычно класс нельзя изменять "на лету"  

В прототипно-ориентированном:  

- формально нет классов  
- объекты создаются из других объектов (клонированием или функцией конструктором)  
- новый объект создается путем копирования прототипа (привязка)  
- наследование (делегирование) работает так - если у объекта нет свойства, он идет к своему прототипу и далее по иерархии прототипов (прототипное наследование)  
- объекты (родительские) можно менять во время работы программы  

---  

В C# есть Рефлексия, но она не может в рантайме поменять класс (добавить поле, интерфейс) и всех его наследников.  
Но есть класс ExpandoObject, который позволяет у одного конкретного объекта что-нибудь поменять.  
В принципе, можно самому руками прописать свой механизм - пусть объект хранит Dictionary методов и их использовать динамически.
